# План: Фильтрация записей на главном экране по цветовым тегам

## Цель

Добавить на главный экран фильтрацию записей по цветовым тегам, где список доступных тегов формируется только из реально существующих записей (только фактически используемые `colorTag`).

## Контекст и ограничения

- Архитектура: MVVM + Clean Architecture, без сетевых вызовов (offline-only).
- Безопасность: не использовать `!!`, только безопасное извлечение значений.
- Логи приложения по умолчанию на русском.
- Новая кнопка открытия фильтров размещается справа от кнопки сортировки в `TopAppBar`.
- Иконка кнопки: стандартная `Icons.Filled.FilterList` (наиболее подходящая по смыслу).
- Источник фильтров: только цвета, реально присутствующие у записей (`item.colorTag != null`).

## Риски и решения

- [ ] **Риск:** изменение фильтрации может сломать текущую логику поиска.
  - **Снижение риска:** отдельные unit-тесты на пересечение поиска и фильтра по цвету.
- [ ] **Риск:** дублирующиеся цвета в меню фильтра.
  - **Снижение риска:** формировать `availableColorTags` через `distinct()`.
- [ ] **Риск:** выбранный цвет устарел после изменений данных.
  - **Снижение риска:** авто-сброс `selectedColorTag` при отсутствии цвета в актуальном списке.
