# План: Фильтрация записей на главном экране по цветовым тегам

## Цель

Добавить фильтрацию по цветовым тегам на главном экране через диалоговое окно. Набор цветов для выбора формируется только из реально существующих `colorTag` в записях. Фильтр должен работать совместно с поиском и сохраняться между перезапусками приложения.

## Контекст и ограничения

- Архитектура: MVVM + Clean Architecture, offline-only.
- Безопасность: не использовать `!!`.
- Логи: на русском.
- Кнопка фильтра находится в `TopAppBar` справа от сортировки.
- Иконка кнопки: `Icons.Filled.FilterList`.
- UI фильтра: диалог с заголовком `Фильтр по цвету`, сетка цветов, кнопки `Применить` и `Сбросить`.
- Визуал выбора цвета в сетке: бордюр по аналогии с экраном создания/редактирования (`CreateEditScreen`).

### 3.2 Диалог фильтра

- [ ] По нажатию на кнопку открывать диалог (`AlertDialog`/эквивалент) с заголовком:
  - `Фильтр по цвету`.
- [ ] В контенте диалога отрисовать сетку всех `availableColorTags`.
- [ ] Клик по цвету в сетке выбирает его (single-select).
- [ ] Выбранный элемент в сетке подсвечивается бордюром (как на Create/Edit экране).

### 3.3 Кнопки `Применить` и `Сбросить`

- [ ] Внизу диалога добавить две кнопки:
  - `Применить`,
  - `Сбросить`.
- [ ] Состояние `Применить`:
  - активно только если выбранный в диалоге цвет отличается от уже применённого фильтра.
- [ ] Состояние `Сбросить`:
  - активно только если уже применён какой-либо `selectedColorTag != null`.
- [ ] Поведение `Применить`:
  - сохранить выбранный цвет в DataStore,
  - применить фильтрацию,
  - закрыть диалог.
- [ ] Поведение `Сбросить`:
  - очистить фильтр (`null`) в DataStore,
  - применить сброс фильтрации,
  - закрыть диалог.

### 3.4 Локализация

- [ ] Добавить строки в `values/strings.xml` и `values-ru/strings.xml`:
  - `Фильтр по цвету` / `Filter by color`,
  - `Применить` / `Apply`,
  - `Сбросить` / `Reset`,
  - `Открыть фильтр` (contentDescription),
  - при необходимости подписи для состояния выбора.

**Критерий завершения этапа:** диалог фильтра работает по описанному UX-сценарию, кнопки имеют корректную активность.

- [ ] Зафиксировать и проверить, что применяется пересечение:

- [ ] Если выбранный цвет исчез из данных (удалили последнюю запись этого цвета), фильтр автоматически сбрасывается на `null`.
- [ ] После авто-сброса UI не должен оставаться в неконсистентном состоянии.

- [ ] При отсутствии результатов из-за поиска/цвета использовать текущее `EmptySearchContent`.
- [ ] (Опционально) добавить отдельную строку под ситуацию «фильтр применён, результатов нет».

**Критерий завершения этапа:** фильтр предсказуемо работает во всех переходных состояниях.
