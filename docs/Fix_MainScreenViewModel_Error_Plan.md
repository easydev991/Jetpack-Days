# План исправления ошибки MainScreenViewModel

## Обзор проблемы

При запуске приложения возникает фатальная ошибка:
```
java.lang.RuntimeException: Cannot create an instance of class com.dayscounter.viewmodel.MainScreenViewModel
Caused by: java.lang.NoSuchMethodException: com.dayscounter.viewmodel.MainScreenViewModel.<init> []
```

Ошибка указывает на то, что ViewModel не имеет публичного конструктора без аргументов, который ожидает ViewModelProvider.

## Анализ ошибки

1. **Тип ошибки**: `NoSuchMethodException`
2. **Класс**: `com.dayscounter.viewmodel.MainScreenViewModel`
3. **Проблема**: Отсутствует публичный конструктор без аргументов (`<init> []`)
4. **Причина**: ViewModelProvider пытается создать экземпляр класса через рефлексию, используя конструктор без аргументов, но такой конструктор отсутствует

## Возможные причины

1. MainScreenViewModel имеет конструктор с аргументами и не предоставляет публичный конструктор без аргументов
2. MainScreenViewModel использует внедрение зависимостей (DI) через конструктор, но не настроено правильное создание экземпляра
3. MainScreenViewModel аннотирован как @HiltViewModel, но Hilt не настроен или не работает

## План решения

### Этап 1: Анализ текущей реализации MainScreenViewModel

1. Найти файл `MainScreenViewModel.kt`
2. Изучить текущую реализацию конструктора
3. Определить зависимости, которые требуются ViewModel

### Этап 2: Исправление конструктора ViewModel

1. Если используется Hilt, убедиться, что:
   - Класс аннотирован как `@HiltViewModel`
   - Все зависимости помечены как `@Inject`
   - В проекте настроены Hilt плагины и зависимости
2. Если используется ручное внедрение зависимостей, реализовать фабричный метод или использовать подход с SavedStateHandle

### Этап 3: Обновление использования ViewModel в UI

1. Обновить вызов `viewModel()` в `MainScreen.kt`, если требуется передать зависимости
2. Убедиться, что все зависимости корректно передаются через SavedStateHandle или через DI

### Этап 4: Тестирование

1. Собрать и запустить приложение
2. Убедиться, что ошибка больше не возникает
3. Проверить функциональность экрана

## Ожидаемый результат

Приложение успешно запускается без ошибок, связанных с созданием экземпляра MainScreenViewModel.
