# Документация: Фильтрация по цветовым тегам на главном экране

## Назначение

Функционал позволяет отфильтровать список событий на главном экране по цветовому тегу.  
Фильтр работает совместно с текстовым поиском и сохраняется между перезапусками приложения.

## Модель данных и персистентность

- Активный фильтр хранится в DataStore как `Int?` (`COLOR_TAG_FILTER_KEY`).
- `null` означает отсутствие фильтра.
- Значение читается в `MainScreenViewModel` через `StateFlow` и влияет на формирование итогового списка.
- При смене фильтра вызывается `setMainScreenColorTagFilter(...)`.
- При рестарте приложения последний выбранный фильтр восстанавливается автоматически.

## Источник цветов для фильтра

- Список цветов формируется только из реально существующих `colorTag` у событий.
- Цвета без привязанных записей не показываются в диалоге.
- Если активный цвет исчезает из набора доступных (например, после удаления/изменения записей), фильтр автоматически сбрасывается в `null`.

## Поведение на главном экране

### TopAppBar

- Кнопка сортировки находится в `navigationIcon`.
- Кнопка фильтра находится в `actions`, справа от сортировки.
- Иконка фильтра показывает состояние:
  - `Icons.Outlined.Palette` при неактивном фильтре;
  - `Icons.Filled.Palette` при активном фильтре.

### Правила видимости кнопки фильтра

- Кнопка показывается, если:
  - `availableColorTags.isNotEmpty()`, и
  - `itemsCount >= 2` или фильтр уже активен (`selectedColorTag != null`).
- При одной записи и неактивном фильтре кнопка скрыта.
- Если у всех записей `colorTag = null`, кнопка скрыта.

## Диалог фильтра по цвету

### Содержимое

- Заголовок: `Фильтр по цвету`.
- Контент: сетка доступных цветов.
- Внизу две кнопки: `Применить` и `Сбросить`.

### Выбор цвета

- Режим выбора: single-select.
- Повторный тап по уже выбранному цвету снимает выбор в черновике.
- Для больших наборов цветов используется вертикальный скролл внутри ограниченной по высоте области.
- Сетка адаптивная по колонкам, элементы не деформируются.

### Локальное состояние диалога

- В диалоге используется черновик `draftSelectedColor`.
- При открытии диалога черновик инициализируется текущим примененным фильтром.
- Черновик не применяется до нажатия `Применить`.

### Кнопки действий

- `Применить` активно только если черновик отличается от текущего фильтра.
- `Сбросить` активно если:
  - есть примененный фильтр, или
  - в черновике выбран цвет.
- Поведение `Сбросить`:
  - при примененном фильтре: сбрасывает фильтр и закрывает диалог;
  - при пустом примененном фильтре, но выбранном черновике: очищает только черновик и оставляет диалог открытым.

## Совместная работа поиска и цветового фильтра

- Итоговый список рассчитывается по принципу `поиск AND цвет`.
- Порядок применения в потоке данных:
  - сортировка;
  - фильтр по поисковому запросу;
  - фильтр по выбранному цвету.
- Изменение любого из входов (`sortOrder`, `searchQuery`, `selectedColorTag`, список записей) приводит к пересчету итогового списка.

## Визуал цветовых тегов

- Для выбора цвета используется общий компонент `SelectableColorTag`.
- При выборе применяется двойная обводка:
  - внешний контур `outline`;
  - внутренний белый контур.
- Толщина обоих контуров одинаковая.
- Актуальные размеры в `dimens.xml`:
  - `color_tag_size = 42dp`;
  - `color_tag_size_small = 16dp`.

## Аналитика

- События пользовательских действий:
  - `sort` (`UserActionType.SORT`) при нажатии на кнопку сортировки в `TopAppBar`;
  - `open_filter` (`UserActionType.OPEN_FILTER`) при нажатии на кнопку открытия диалога фильтра.
- Логирование выполняется через `AnalyticsService` с событием `AnalyticsEvent.UserAction(...)`.

## Пустые состояния

- Для пустого результата поиска используется `EmptySearchContent`.
- Цветовой фильтр не вводит отдельный экран пустого состояния и опирается на текущее поведение пустого поиска.

## Покрытие тестами

- Unit-тесты:
  - сохранение/чтение фильтра в DataStore;
  - фильтрация по цвету в ViewModel;
  - совместная работа поиска и цвета;
  - авто-сброс недоступного активного цвета;
  - значения аналитических action (`AnalyticsEventTest`).
- UI-тесты:
  - сценарии диалога `Применить/Сбросить`;
  - состояние кнопок при изменении/сбросе черновика;
  - корректное отображение/взаимодействие с сеткой цветов.

## Рекомендации для развития

- Если будет добавлен color picker с произвольными цветами:
  - оставить источником списка только реально используемые в записях цвета;
  - проверить масштабируемость диалога для сотен цветов (виртуализация/группировка).
- При добавлении новых аналитических событий:
  - расширять `UserActionType`;
  - обновлять `AnalyticsEventTest`;
  - синхронизировать документацию в этом файле.
- При изменении UX фильтра:
  - сохранять принцип отложенного применения через черновик;
  - отдельно проверять совместимость с поиском и персистентностью фильтра.
